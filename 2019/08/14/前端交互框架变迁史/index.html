页面内容越来越丰富给我们带来的一些问题

  1. 页面内容多而复杂，如何保证开发效率？使用前端框架（前端开发框架范畴）
  2. 页面内容多而复杂，项目的管理和维护如何去做？模块化和组件化思想（模块化和组件化开发范畴）
  3. 如何保证尽快将网页内容显示给用户？使用异步编程（资源异步加载范畴）
  4. 如何控制页面资源的大小以保证页面快速展示？对资源进行优化处理（前端优化范畴）
  5. 如果重复打开的页面，能够不再向服务端发送请求？合理地利用文件缓存（缓存范畴）
  6. 页面地址在移动端打开，该如何处理？使用UI和内容更简洁的页面（响应式范畴）

1.借助符合特定场景的前端框架来提升开发效率。例如对常用的DOM进行高效封装，大大简化开发工作量，提高效率

2.用模块化和组件化的思路来管理项目，采用代码管理中分治的思想，将复杂的代码结构拆分成多个独立、简单、解耦合的解构或文件分开管理，使项目结构更加清晰。

3.可以通过异步的方式来实现，将一部分内容先展示给用户，然后根据用户的操作，异步加载用户需要的其他内容，避免用户长时间等待

4.考虑图片的优化处理，如使用更高压缩比webp格式的图片，在图片质量不降低的情况下，大幅度减少图片的网络流量消耗，提高图片加载速度

5.浏览器默认支持文件缓存，对于一段时间内浏览器的重复请求，服务器可能会返回HTTP的304状态码或者不发送请求，让浏览器直接从本地读取内容

6.移动端浏览器整体性能不如桌面浏览器，使用移动端浏览器打开一个web页面时，会打开一个更加简洁的页面

前端开发模式变迁

  1. 静态黄页时期
  2. 服务器组装动态网页数据时期
  3. 后端为主的MVC模式时期
  4. 前后端分离方案开发时期
  5. 纯前端MV*为主与中间层直出的时期
  6. 前端虚拟DOM、MNV*模式，前后端同构的开发时期

演变总结

  • 持续在以效率和质量为最终导向的道路上探索前进
    • 效率：从前后端分离到各种封装的前端框架，都在解决前端编程的效率问题
    • 质量
      • 性能：Virtual DOM和MNV*交互模式等的实现思路就是为了解决前端交互性能问题
  • 前端工程、自动化构建、组件化、优化等技术解决方案的出现，也是为了效率和质量提升
