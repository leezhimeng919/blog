链表LinkedList

  • 链表是一组节点组成的集合。除了最后一个节点每个节点都有一个后继节点
  • 链表的构造函数：Node类和LinkedList类
  • 链表的方法：
    • 增加新元素 ：append(element)
    • 插入新元素 : insert(element, position)
    • 按位删除元素 : removeAt(position)
    • 按值删除元素 : remove(element)
    • 查找元素位置 : indexOf(element)
    • 查看链表长度 : size()
    • 查看head元素 : getHead()

web前端交互框架发展历史

  • web前端起始阶段，没有dom交互，静态文本放到服务器目录下，除了页面跳转没有别的操作
  • web前端的早期dom交互时代，利用数据库技术将数据库中的数据展示出来，用户可以单向订阅网站内容，刷新页面提交操作
  • DOM交互式页面：ajax时代（高效的处理dom操作和事件绑定）
    • DOM加载
    • JavaScript脚本加载
    • 脚本执行发送请求
    • DOM操作进行数据渲染
    • 事件绑定
    • 有点：高效的处理dom操作和事件绑定，带来了效率的提升
    • 缺点：业务和页面结构的复杂性提升，增加了维护管理的难度
  • MV*模式
    • 针对dom交互式页面的缺点，将dom相关内容抽象成三个部分

网站交互方式

  • 多页MPA
    • 缺点
      • 页面切换慢，等待时间过长（用户体验一般，每次跳转都要刷新和重新加载渲染）
        • 因为每次跳转都需要发出一个http请求
      • 前后端耦合在一起，开发和维护效率低
    • 优点
      • SEO
        • MPA的每个页面的所有内容都在html中，seo效果好
      • 首屏快
        • 只请求了一个页面的资源
  • 单页SPA
    • 优点
      • 局部刷新，用户体验好
      • 前后端分离
      • 组件化
        • 不再以页面为单位，更多地采用组件化的思想，便于修改和调整
      • 共享
        • API多端共享，减少服务端工作量，将容易变化的UI部分前置到了多端
        • 组件多端共享，节约资源，快速迭代
    • 缺点
      • 首屏慢
        • 加载所有静态资源
      • SEO不行
        • 单页面中大量数据是通过异步加载过来的，很难被搜索引擎搜到
      • 开发难度高
        • 类似于一个运行在浏览器中的桌面应用，对js、组件化、设计模式要求高
      • 兼容性
        • 无法兼容低版本浏览器

RESTful规范接口风格

  • 使用URL来定位资源，使用HTTP方法来描述操作
    • 接口名用名词，HTTP请求类型区分动作(增删改查)
    • 版本号写在URL中，更加标准的restful将版本号写在HTTP请求头的Accept中，但查看不太方便
    • path/v1/books GET 获取书籍信息，版本一
