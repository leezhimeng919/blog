到底什么是提升？是指声明提升么？

回答

本题是变量声明提升的问题，

我们都知道var声明能提升是因为JavaScript引擎会做如下操作

ES6之后新增let

问题出现了，为什么name = 1会报引用错误？引用错误指的是当一个不存在的变量被引用时报错

请问：既然let没有提升，所以JavaScript引擎就是常规的一行一行的解析。当JavaScript引擎还没有读到let name的时候。name = 1怎么知道name被let声明了？

所以我理解的是：let将name这个变量提升到了这个块作用域的最顶端
(javascript引擎最终如何编译的我不知道，反正不会是类似于var name的let name)，告诉了这个块作用域的所有用到name的地方，我将在第X行正式声明，在我正式声明之前，谁都不准用。所以形成了所谓的暂时性死区。所以我理解的是
let声明的变量能提升，只是这个变量没有被初始化为undefined即没有被真正的声明。不然这个块作用域当还没读到let的时候是如何知道这个块作用域中有let的存在。

这是我的一点理解和论据。我知道国内很多教材都斩钉截铁的说let不存在提升，至于为什么也没说的很清楚。但我也一直存在这个疑问，但是我觉得如果在面试的时候还是直接说let不能提升吧，因为默认的提升就是变量声明提升，即变量是可用的。let是属于声明提升，声明的是变量在正式声明之前不可用，形成了所谓的暂时性死区
