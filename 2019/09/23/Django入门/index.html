什么是Django

  • Django是一个高级的Python网络框架，快速开发安全和可维护的网站

    完备性

  • 功能完备，开箱即用

    通用性

  • 可以与任何客户端框架一起工作，并可以提供任何格式的内容

    安全性

  • 提供了一些安全的方法避免常见的错误
  • Django可以防范包括SQL注入、XSS跨站点脚本、CSRF跨站点请求伪造和点击劫持等漏洞

    可扩展

  • 基于组件的无共享架构（架构的每一部分互相独立）

    可维护性

  • 代码遵循设计原则和模式

    灵活性

  • 不受平台限制，基于Python

Django代码的特征

传统数据驱动网站

  1. WebAPP等待浏览器发送HTTP请求
  2. 接收到请求后，根据URL和POST/GET中的信息确定需要的内容
  3. 根据需要，从数据库读取或写入信息，或执行满足请求所需的其他任务。
  4. WebAPP返回对浏览器的响应，通过将请求到的数据插入HTML模板中的占位符来动态创建HTML页面。

Django构建的WebAPP

  • 将上述每个步骤的代码分组到单独的文件中
    • URLs(urls.py)
      • URL映射器用于根据请求URL将HTTP请求重定向到相应的视图
      • URL映射器能够匹配出现在URL中的字符串或数字的特定模式，将其作为数据传递给视图
    • View(views.py)
      • 视图是一个请求处理函数，接收HTTP请求并返回HTTP响应。
      • 视图通过模型访问数据（满足请求需要的数据），并将响应格式委托给模板。
    • Models(models.py)
      • 模型是定义APP数据结构的Python对象，- 提供在数据库中管理和查询记录的机制
    • Templates(<filename>.html)
      • 模板是定义文件的结构或布局的文本文件，用于表示实际内容的占位符。
      • 一个视图可以使用HTML模板，动态地创建HTML页面模型
      • 模板可以用来定义任何类型的文件结构(不一定是HTML)
  • MVT架构：模型视图模板架构
