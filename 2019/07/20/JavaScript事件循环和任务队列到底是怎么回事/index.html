JavaScript事件循环和任务队列到底是怎么回事

发表于 2019-07-20 | 更新于 2019-09-05 | 分类于 JavaScript | 阅读次数：

事件循环

什么是事件循环

一种解决JavaScript单线程运行时不会阻塞的一种机制，是异步的原理，浏览器环境和Node环境的事件循环不太一样。

栈和队列

  • 栈是只能在一端（表尾）进行插入或删除的线性表，以先进后出的方式存储数据
  • 队列是一种在一端(表头)删除，另一端（表尾）插入操作的线性表，以先进先出的方式存储数据

    任务队列和调用栈

  • 任务队列是用来处理异步任务的，分为两种宏任务队列和微任务队列

JavaScript中的任务

一种宏任务MacroTask，也叫Task，一种叫MicroTask

宏任务

  • script全部代码
  • setTimeout
  • setInterval
  • setImmendiate
  • I/O
  • UI Rendering

微任务

  • Process.nextTick （Node独有）
  • Promise
  • Object.observe （已废弃）
  • MutationObserver

浏览器中的事件循环

JavaScript中有一个主线程和调用栈，所有的任务都会被放到调用栈中等待主线程执行

调用栈

函数执行时，会被添加到调用栈的顶部，函数执行完后会从栈顶弹出，直至栈被清空

同步任务

同步任务会在调用栈中按照顺序等待主线程依次执行

异步任务

  1. 在异步任务有了结果后，将注册的回调函数放入到任务队列中
  2. 等待主线程空闲时间（调用栈被清空），将被读取到栈内等待主线程的执行

事件循环的进程模型

  1. 选择当前要执行的宏任务队列，选择宏任务队列中最先进入的宏任务，如果任务队列为空，则执行跳转到微任务的执行步骤
  2. 将事件循环中的任务设置为已选择任务
  3. 执行任务
  4. 将事件循环中当前运行任务设置为空
  5. 将已经运行完成的任务从任务队列中删除
  6. 更新界面渲染
  7. 返回第一步

微任务的执行步骤

  1. 设置微任务检查点标志为true
  2. 当事件循环微任务执行不为空时，选择一个最先进入微任务队列的微任务
  3. 将事件循环的微任务设置为已选择的微任务
  4. 运行microtask
  5. 将已经执行完成的microtask为空，移除微任务队列中的微任务
  6. 清理IndexDB事务
  7. 设置进入微任务检查点的标志为false
